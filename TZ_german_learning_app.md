# ТЗ: Приложение для закрепления немецкого языка (учебные фото → упражнения)

## 0. Контекст для разработки

Это ТЗ предназначено для Claude Code. Проект нужно поднять в Docker, по аналогии с другими проектами на этом же сервере (n8n, open-webui уже крутятся рядом — см. docker-compose соглашения, порты не должны пересекаться с занятыми: 3000, 8080, 5678 уже заняты).

Деплой: отдельный поддомен `translate.seoshkin.tools`. Не интегрируется с существующей Drupal CRM — полностью отдельный сервис.

---

## 1. Назначение

Приложение помогает школьнику закреплять материал уроков немецкого языка (уровень: самое начало, первые уроки, A1). После урока пользователь:
1. Фотографирует страницы учебника и рукописные записи в тетради
2. Опционально прикладывает аудиозапись урока
3. Получает автоматически сгенерированные упражнения для закрепления
4. Проходит их сразу или позже — приложение напоминает по алгоритму интервального повторения

Есть 10 уже отснятых уроков (фото учебника + тетради) — их нужно будет загрузить и обработать первыми, это фактически тестовые данные.

## 2. Пользователи

- Один класс: учитель/родитель (админ, вероятно это сам пользователь) + несколько школьников
- Регистрация — максимально простая: email + пароль, без подтверждения почты на старте, без соцсетей. Один "класс" = группа пользователей, привязанных к одному учителю/владельцу.
- Ролей на MVP две: `owner` (загружает уроки, видит всех) и `student` (проходит упражнения, видит свой прогресс).

## 3. Архитектура (высокий уровень)

```
┌─────────────────┐
│  PWA Frontend    │  React/Vue + Service Worker, работает как приложение
│  (Mac/Android)   │  на десктопе и мобиле через "добавить на экран"
└────────┬─────────┘
         │ REST/HTTPS
┌────────▼─────────┐
│  Backend API      │  Node.js (Fastify/Express) или Python (FastAPI)
│                    │  - auth (JWT, простая)
│                    │  - upload фото/аудио
│                    │  - вызовы Claude API (vision + генерация упражнений)
│                    │  - вызовы Whisper API (транскрибация)
│                    │  - SRS-логика (интервальное повторение)
└────────┬─────────┘
         │
┌────────▼─────────┐      ┌──────────────────┐
│  PostgreSQL        │      │  Файловое хранилище│
│  (уроки, слова,     │      │  (фото/аудио,      │
│   упражнения,       │      │   volume в Docker) │

└──────────────────┘

Внешние API:
- Anthropic Claude API (vision + генерация упражнений) — ключ уже есть
- OpenAI Whisper API (транскрибация аудио) — ключ уже есть
```

Всё разворачивается через `docker-compose.yml`: контейнер backend, контейнер frontend (или отдаётся статикой из backend/nginx), контейнер Postgres, volume для загруженных файлов.

## 4. Основной пользовательский флоу

### 4.1 Загрузка урока
- Экран "Новый урок": дата, название/тема (опционально)
- Загрузка нескольких фото за раз (галерея или камера с телефона/веба)
- Опционально — загрузка аудиофайла или запись прямо в браузере (MediaRecorder API)
- Кнопка "Обработать урок"

### 4.2 Обработка (бэкенд)
1. **Фото → распознавание**: каждое фото отправляется в Claude API (vision) с промптом вида:
   > "Это фото страницы учебника/тетради школьника, изучающего немецкий язык на уровне A1. Распознай весь текст, включая рукописный (пользователь старается писать разборчиво). Выдели: (а) новые немецкие слова с переводом, если он есть на странице, (б) грамматические конструкции/правила, (в) примеры предложений. Верни строго в JSON."
   - Структурированный вывод: `{ words: [...], grammar_points: [...], example_sentences: [...], raw_text: "..." }`
2. **Аудио → текст**: если есть аудиозапись, отправляется в Whisper API (транскрипция). Учитывать, что запись может содержать смесь немецкого и русского (объяснения учителя) — Whisper обычно неплохо справляется с code-switching, но нужно на это рассчитывать при парсинге.
3. **Объединение**: результаты по всем фото + аудио одного урока объединяются в единый "конспект урока" (список слов, грамматика, фразы) — тоже через Claude API, отдельным вызовом (сведение дублей, нормализация форм слов).
4. **Генерация упражнений**: на основе объединённого конспекта Claude API генерирует набор упражнений (см. п. 5), возвращая структурированный JSON.
5. Все данные сохраняются в БД, привязанные к уроку и к пользователю (владельцу класса).

### 4.3 Прохождение упражнений
- Экран со списком доступных на сегодня карточек/упражнений (по алгоритму SRS)
- Прохождение — простой интерфейс (см. типы упражнений ниже)
- Ответ пользователя оценивается: либо локально (точное сравнение строк с учётом опечаток), либо через Claude API для более гибкой оценки (например, для составления предложений)
- Результат обновляет статус слова/упражнения в SRS

### 4.4 Личный словарь
- Список всех слов, извлечённых из уроков
- Статус: новое / изучается / выучено
- Возможность вручную пометить слово как "уже знаю" (пропустить из повторения)

## 5. Типы упражнений (MVP)

| Тип | Описание | Приоритет |
|---|---|---|
| Флеш-карты | слово (нем.) ↔ перевод, с оценкой "помню/не помню" | MVP |
| Заполнение пропуска | предложение с пропущенным словом из урока | MVP |
| Выбор формы | артикль/окончание/спряжение — множественный выбор | MVP |
| Диктант по аудио | прослушать фрагмент из Whisper-транскрипции, написать услышанное | v1 (сразу, раз Whisper делаем) |
| Устный ответ + проверка | наговорить фразу, Whisper транскрибирует, сравнение с эталоном на уровне слов (что сказано верно/пропущено) | v1, упрощённая версия — без фонетической оценки произношения |
| Составление предложения из слов | drag-n-drop или ввод текста | v2 |

Полноценная оценка произношения (фонетика, акцент) — вынесена в последний этап, отдельный модуль, не в MVP.

## 6. Интервальное повторение (SRS)

- Алгоритм: SM-2 (как в Anki) — простой, хорошо задокументирован, легко реализовать самостоятельно без внешних библиотек.
- Для каждого слова/упражнения хранится: `easiness_factor`, `interval`, `repetitions`, `next_review_date`.
- Ежедневный экран "к повторению сегодня" — сортировка по `next_review_date <= today`.

## 7. Модель данных (черновик)

```
users
  id, email, password_hash, role (owner/student), class_id, created_at

lessons
  id, owner_id, date, title, created_at

lesson_media
  id, lesson_id, type (photo/audio), file_path, processed (bool), raw_extraction (jsonb)

words
  id, lesson_id, user_id, word_de, translation_ru, example_sentence,
  easiness_factor, interval_days, repetitions, next_review_date, status (new/learning/known)

grammar_points
  id, lesson_id, description, example

exercises
  id, lesson_id, type, payload (jsonb: вопрос/варианты/правильный ответ), created_at

exercise_attempts
  id, exercise_id, user_id, user_answer, is_correct, attempted_at
```

## 8. Технический стек (рекомендация)

| Компонент | Технология |
|---|---|
| Backend | Node.js + Fastify (или Python + FastAPI — на усмотрение Claude Code, главное — просто и без лишних абстракций) |
| Frontend | React + Vite, PWA (vite-plugin-pwa), минимум зависимостей |
| БД | PostgreSQL |
| Хранение файлов | локальный volume в Docker (папка `/data/uploads`), без облака на старте |
| Auth | JWT, простой email+пароль (bcrypt для хэша) |
| Vision/генерация упражнений | Anthropic Claude API (ключ передаётся через переменную окружения `ANTHROPIC_API_KEY`) |
| Транскрибация аудио | OpenAI Whisper API (`OPENAI_API_KEY`) |
| Контейнеризация | Docker + docker-compose, отдельные сервисы: `backend`, `frontend` (или backend отдаёт статику), `db` |
| Reverse proxy | предполагается, что на сервере уже есть nginx/traefik для других проектов — новый сервис нужно завести на поддомен `translate.seoshkin.tools`, проброс порта уточнить при деплое |

## 9. Порты и деплой

- На сервере уже заняты порты 3000 (open-webui), 5678 (n8n) — выбрать свободный порт для backend (например, 8090) и не пересекаться.
- `.env` файл с ключами (`ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, `DATABASE_URL`, `JWT_SECRET`) — не коммитить в git, добавить в `.gitignore`.
- `docker-compose.yml` в корне проекта — по аналогии с существующими проектами на сервере.

## 10. Приоритизация

### MVP (первая версия — сделать в первую очередь)
- Регистрация/логин (owner + student)
- Загрузка фото урока → Claude vision → извлечение слов/грамматики
- Загрузка аудио → Whisper → транскрипция → добавление в конспект урока
- Генерация упражнений: флеш-карты + заполнение пропуска + выбор формы
- SRS (SM-2) для слов
- Простой личный словарь со статусами
- Импорт и обработка 10 уже готовых уроков (тестовые данные)

### v1 (сразу после MVP)
- Диктант по аудио
- Устный ответ с текстовым сравнением (без фонетики)
- Статистика прогресса (сколько слов выучено, календарь повторений)

### v2 (позже)
- Составление предложений из слов (drag-n-drop)
- Более тонкая генерация упражнений под слабые темы ученика
- Полноценная фонетическая оценка произношения (отдельный модуль, вероятно сторонний API типа Azure Pronunciation Assessment)

## 11. Открытые вопросы для уточнения в процессе разработки

1. Backend на Node.js или Python — если нет предпочтения, Claude Code выбирает сам исходя из простоты.
2. Structured output от Claude API — использовать ли `tool_use` для гарантированного JSON или просто промптить "верни только JSON" (проще для MVP, но чуть менее надёжно).
3. Хранить ли оригинальные фото после обработки (для проверки качества распознавания) или удалять после извлечения текста — рекомендация: хранить, диск дешёвый, полезно для отладки OCR.
4. Одна БД на весь проект или сразу продумывать мультитенантность (несколько классов/учителей) — на старте не нужно, только один owner + его ученики.
