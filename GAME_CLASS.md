# 🎮 Игра «Класс говорит» (живой урок фразами)

## Идея Павла
В конце урока вместо 5 словоформ — **25–35 РАЗНЫХ предложений** из слов урока. Каждому ученику приходят СВОИ фразы (не одна и та же на всех, как в классе), с переводом на ЕГО локаль. Читаем по очереди — каждый своё. Так учим немецкий лучше. Учитель — ведущий игры. Наборы фраз копятся в разговорник (тематично).

## Как устроено

### 1. Генерация (после урока, кнопка «🎮 Собрать игру класса»)
- Учитель (или авто после обработки урока) жмёт «Собрать игру» на уроке.
- ИИ по словам урока делает N фраз (N ≈ ученики × 3–5, или фикс 25–35): короткие A1, каждая использует лексику урока, **разные** между собой. Часть — вопросы, часть — ответы/утверждения. Тематично.
- Каждая фраза сразу переводится на **все 10 локалей** (JSONB), чтобы каждый ученик видел свой перевод.

### 2. Раздача
- Фразы раздаются ученикам **по кругу** (round-robin): у каждого — личная колода уникальных фраз, ни у кого не совпадает.
- Каждому — **push**: «🎮 Игра по Уроку X готова! Твои фразы ждут. Начнём, когда скажет учитель.»

### 3. Форматы (выбор учителя)
- **Эстафета:** читают по очереди вслух. Приложение показывает, чей ход, крупно фразу + перевод + 🔊 озвучку. Учитель жмёт «Следующий».
- **Вопрос-ответ:** фразы парами — ученик A читает вопрос, ученик B читает ответ. Пары назначает игра.

### 4. Экран ученика
- Немецкая фраза крупно + перевод на его язык + 🔊.
- Когда его ход — подсветка «Твой ход, читай!» + кнопка «✅ Прочитал».
- (Опционально) запись голоса → Whisper проверяет произношение (переиспользуем тренер).

### 5. Учитель — ведущий
- Видит ВСЕ фразы и кто на каком ходе, ведёт темп («Следующий»), видит кто прочитал.
- Может добавить свои фразы, перемешать, повторить круг.

### 6. В разговорник
- Весь набор фраз → в разговорник каждого ученика (тематическая подборка урока), с переводом на его локаль.

## Данные
- `class_games` (id, lesson_id, owner_id, format, status, created_at).
- `class_game_lines` (id, game_id, ord, sentence_de, translations JSONB, role[question|answer|statement], assigned_user_id, read_at).

## API
- `POST /api/class-games` {lesson_id, count, format} → ИИ-генерация фраз + перевод 10 языков + раздача ученикам + push.
- `GET /api/class-games/:id` → учитель видит все + текущий ход; ученик — свои + чей сейчас ход.
- `POST /api/class-games/:id/next` → следующий ход (учитель).
- `POST /api/class-games/:id/lines/:lid/read` → отметка «прочитал».
- `POST /api/class-games/:id/to-phrasebook` → сохранить набор в разговорники.

## Плюшки учителю (чтобы хотелось пользоваться)
- **Роль ведущего** — управляет темпом, видит класс живьём, решает кто читает.
- **Живая карта класса** — кто активен, кто сколько слов выучил, кто буксует (тепловая карта).
- **Учитель как персонаж** — в игре/тренере у него свой аватар, он «в игре» вместе с классом.
- **Ачивки и награды** — учитель выдаёт значки ученикам («лучшее произношение», «прочитал все»).
- **Домашка-игрой** — задал набор → пуш классу → уведомление учителю, когда все прошли.
- **Свои фразы** — учитель добавляет собственные предложения в набор.
- **Отчёт-геймификация** — прогресс класса как «уровни», а не сухая таблица.

## Этапы внедрения
1. **MVP:** генерация набора фраз из урока (25–35, мультилокаль) + раздача по ученикам + экран ученика (моя фраза + перевод + 🔊) + сохранение в разговорник. Асинхронно (без live-хода).
2. **Live-режим:** учитель ведёт ход в реальном времени (WebSocket / polling), эстафета/вопрос-ответ.
3. **Плюшки учителю:** карта класса, ачивки, свои фразы.
