# 📊 Отчёт по уроку — дизайн (п.1 «Класс говорит v2»)

Дата: 2026-07-16 · Автор: Opus (с Павлом) · Статус: согласован, готов к плану

## Контекст и решение

Исходный п.1 из resume-state — «Класс говорит v2»: словоформы, N повторов на слово,
проговорить вслух, лист на печать, отчёт «кто/группа на чём завис».

**Что решили на брейншторме:**
- Новый режим «словоформы + N повторов + проговорить вслух» — **отклонён как дубль**:
  это на 90% повторяет уже готовые «Проговори слова» (speech + Whisper), флеш-карты,
  «Добавь букву», AI-тренер по словам урока. Строить ещё один режим = мусор в системе.
- **Печать листа A4** — закрыта параллельно Fable (её задача), Opus не трогает.
- Реальная новая ценность, которой в системе нет → **📊 Отчёт по конкретному уроку**:
  «кто из класса завис и где буксует вся группа», в разрезе одного урока.

Итог: п.1 = **только Отчёт по уроку**. Ноль пересечений с Fable, ноль дублей.

## Цель

Учитель выбирает урок и сразу видит: (А) на каких словах/типах упражнений буксует
**вся группа**, (Б) как идёт каждый **ученик** по этому уроку и кто завис. Опора — данные,
которые уже собираются (`exercise_attempts`, `user_word_status`). **0 новых таблиц.**

## Определения (формулы «завис»)

- **Слово завязло у ученика** — по упражнениям этого слова у ученика ≥3 попытки И
  точность <50%; ЛИБО статус слова ≠ `known` при ≥4 попытках.
- **Группа буксует на слове** — суммарно по классу ≥3 попытки И высокий % ошибок
  (та же логика, что текущий `hardestWords` в `analytics.js`, но scope = слова этого урока).
- **Не приступил** — ученик школы владельца без единой попытки по упражнениям урока.

## Точка входа (UI)

1. Кнопка **«📊 Отчёт»** на карточке урока у учителя (`LessonCard`, рядом с 🎮/🧩/✨,
   только для `role === 'owner'`) → переход на `/lesson-report/:id`.
2. На странице `/analytics` (`TeacherAnalytics.jsx`) — селектор урока сверху, ведёт туда же.
   Вторичный вход; главный — карточка урока.

## API

`GET /api/analytics/lesson/:lessonId` — owner-only.
- preHandler `fastify.authenticate`; проверка `request.user.role === 'owner'`.
- Проверка владения: урок существует и `lessons.owner_id = request.user.id` (иначе 403/404).
- Scope учеников — школа владельца: `users.school_id = <owner.school_id> AND role='student'`.

Ответ (JSON):
```
{
  lesson: { id, title },
  totals: { students_touched, students_total, attempts, accuracy },
  group:  {
    hardWords: [ { word_de, translation, attempts, wrong_pct, students_stuck } ],  // Блок А
    byType:   [ { type, attempts, accuracy } ]                                     // Блок А
  },
  students: [                                                                      // Блок Б
    { id, name, attempts, accuracy, known, learning, stuck_words, last_active,
      stuck_list: [ { word_de, translation, wrong_pct } ] }  // для раскрытия по тапу
  ],
  notStarted: [ { id, name } ]                                                     // Блок Б низ
}
```

### Запросы (по образцу существующего `analytics.js`)

Все — с фильтром `e.lesson_id = $lessonId` дополнительно к `l.owner_id = $ownerId`.

- **hardWords** — как `hardest`, но `WHERE e.lesson_id = $2`; плюс подсчёт `students_stuck`
  (число учеников, у кого по слову ≥3 попыток и точность <50%).
- **byType** — как `byType`, но `WHERE e.lesson_id = $2`.
- **students** — как `attempts`+`wordStatus` в overview, но:
  - попытки/точность — только по `e.lesson_id = $2`;
  - known/learning — только по словам этого урока (`w.lesson_id = $2`);
  - `stuck_words` — число завязших слов ученика по формуле выше.
- **notStarted** — ученики школы владельца (`u.school_id = owner.school_id AND u.role='student'`),
  которых НЕТ среди `students` (нет попыток по этому уроку). Owner-аккаунты исключаются
  (совмещённый аккаунт Павла не должен попадать в список учеников).

## Фронт

Новая страница `frontend/src/pages/LessonReport.jsx` + роут `/lesson-report/:id`:
- Шапка: название урока + сводка (учеников трогало / всего, средняя точность).
- **Блок А «🔥 Группа буксует здесь»**: список трудных слов (слово + перевод + % ошибок +
  «N учеников завязло»); мини-разрез по типу упражнения (точность на flashcard/fill_blank/…).
- **Блок Б «👤 По ученикам»**: карточки/строки учеников (имя · точность · знаю/учу ·
  зависших слов · последняя активность); тап по ученику раскрывает его трудные слова урока
  (из того же ответа — без доп. запроса). Внизу сворачиваемый «⏳ Не приступили».
- Мультилокаль: все подписи через существующий i18n (10 языков); слова — `word_de` + перевод.
- Стиль — как `TeacherAnalytics.jsx` (переиспользуем визуальные паттерны/компоненты).

## Что сознательно НЕ делаем (границы)

- Печать листа урока — Fable (готово/деплоится).
- Новый режим со словоформами/N повторов — отклонён (дубль).
- Live-ход игры, ачивки, карта класса, экспорт PDF, гейт «пройти самостоятельно» — не в этом п.1.
- Никаких новых таблиц/миграций; тяжёлых вычислений на клиенте нет — агрегаты считает БД.

## Проверка после внедрения

- Отчёт открывается с карточки урока и из `/analytics`; недоступен ученику (403).
- На демо-данных: трудные слова урока совпадают с ручной проверкой по `exercise_attempts`.
- «Не приступили» показывает только учеников школы владельца, без owner-аккаунтов.
- Урок без активности → пустые блоки с понятной заглушкой, без ошибок.
- Подписи локализованы (проверить на 2-3 локалях).
