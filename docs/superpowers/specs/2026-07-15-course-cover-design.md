# Обложка курса — дизайн

## Контекст

German-learning PWA. Курсы (`courses`) — контейнер для уроков (`/courses`,
`/courses/:id`). Сейчас у курса только `title`/`description`, никакой картинки.
Учитель хочет загружать обложку учебника (фото с телефона/диска), чтобы курсы
в списке выглядели как книжная полка.

## Цель

Дать учителю (`role === 'owner'`) загрузить/сменить/убрать обложку курса —
отдельное новое фото, не выбор из уже загруженных сканов уроков (осознанно
решено проще для первой версии — см. «Альтернативы»).

## Данные

Миграция: `ALTER TABLE courses ADD COLUMN cover_image_url TEXT`. Nullable,
никаких данных не переносим, обратно совместимо.

## Бэкенд (`backend/src/routes/courses.js`)

Два новых роута, оба `preHandler: [fastify.authenticate]` +
`role !== 'owner' → 403` (тот же паттерн, что у остальных мутаций курса).

### `POST /api/courses/:id/cover`

- Multipart, один файл (`request.parts()`, как в
  `POST /api/lessons/:id/media`, `backend/src/routes/lessons.js:298-329`).
- Валидация: `mimetype` должен начинаться с `image/`, иначе 400.
- Обработка через `sharp` — новая функция `saveCourseCover(buffer, courseId)`
  в `backend/src/services/imageOptimize.js` (рядом с существующей
  `saveOptimizedImage` для картинок слов, тот же модуль/паттерн):
  - `fit: 'cover'` (обрезка по рамке, не letterbox — так обложка книги
    заполняет карточку без пустых полей), целевой размер ~480×640, webp q82.
  - Один размер (без `_sm`-варианта как у слов) — курсов мало (десятки, не
    тысячи), доп. srcset не оправдан.
  - Сохранить в `uploads/course-covers/course_<id>.webp` (перезапись при
    повторной загрузке — простое имя файла на курс, не нужно чистить старый
    файл отдельно).
- `UPDATE courses SET cover_image_url = $1 WHERE id = $2 AND owner_id = $3
  RETURNING *`.
- Ответ: обновлённый курс (как у `PATCH /api/courses/:id`).

### `DELETE /api/courses/:id/cover`

- `UPDATE courses SET cover_image_url = NULL WHERE id = $1 AND owner_id = $2`.
- Файл на диске не обязательно удалять (не критично для этой версии — не
  усложняем).
- Ответ: 204.

### Существующий `GET /api/courses`

Добавить `c.cover_image_url` в `SELECT` (`backend/src/routes/courses.js:12-24`).
Никаких других изменений в этом роуте.

## Фронтенд

### `frontend/src/pages/CourseView.jsx` (`/courses/:id`)

- Наверху страницы (над/рядом с заголовком курса) — блок обложки:
  портретная картинка (или плейсхолдер, см. ниже), кликабельная ТОЛЬКО для
  `owner`. Клик → скрытый **один** `<input type="file" accept="image/*">`
  БЕЗ атрибута `capture` (не `multiple` — обложка одна). На мобильных без
  `capture` браузер сам предлагает выбор «Камера / Галерея» — двух отдельных
  кнопок (как в задаче A для LessonList) здесь не нужно: там два входа были
  нужны, т.к. один `input` был общий на два РАЗНЫХ источника (учебник/тетрадь);
  здесь источник один, так что достаточно одного input с системным выбором. →
  `POST /courses/:id/cover` (multipart, как `uploadMedia` в
  `LessonList.jsx:73-86` — тот же паттерн через `uploadFiles` из
  `api/client.js`) → обновить `course.cover_image_url` в состоянии.
- Рядом (или по ховеру/долгому тапу) — маленькая кнопка «Убрать обложку»,
  видна только если обложка уже есть, дергает `DELETE .../cover`.
- Студент видит обложку как есть, без возможности менять (не кликабельна).

### `frontend/src/pages/CourseList.jsx` (`/courses`)

- Карточка курса (сейчас — просто текстовый блок, `CourseList.jsx:~72-82`)
  получает портретную обложку сверху (высота ~150-180px в сетке карточек,
  `object-fit: cover`, скруглённые верхние углы в цвет карточки), под ней —
  существующий текстовый блок (title/description/счётчики) без изменений.
- Плейсхолдер (когда `cover_image_url` пуст): не пустое место — красивая
  заглушка на том же месте: мягкий градиент (в акцентных цветах темы,
  `var(--accent-soft)`→`var(--surface-2)`) + иконка книги (📕/📗 — эмодзи,
  как остальной проект) + при желании первая буква названия курса крупно.
  Плейсхолдер — переиспользуемый компонент, чтобы `CourseView.jsx` тоже мог
  им пользоваться (единый визуальный язык).

## Ошибки

- Не тот MIME-тип / файл не выбран → `alert()` с понятным текстом (как
  везде в проекте, никакого нового паттерна уведомлений).
- 403 при попытке загрузить не-owner'ом — фронт не должен показывать
  кликабельную обложку студенту вообще (не полагаемся только на 403 с бэка).
- Ошибка сети при загрузке → `alert('Ошибка загрузки: ' + e.message)`,
  тот же паттерн, что в `LessonList.jsx:78-80`.

## Альтернативы (рассмотрены, отклонены)

- **Выбор обложки из уже загруженных сканов урока** — удобнее в моменте
  (фото учебника часто уже есть среди сканов), но требует UI выбора среди
  фото уроков курса и усложняет первую версию. Отклонено сейчас, можно
  добавить отдельной доработкой позже, если понадобится — не блокирует
  текущий дизайн (новое поле `cover_image_url` не мешает добавить это потом).
- **Хранить картинку как base64 в БД** — просто, но раздувает БД и не даёт
  кэширования браузером/CDN так, как раздача статики через `/uploads/`
  (уже есть инфраструктура, см. `saveOptimizedImage`) — отклонено, не
  соответствует существующему паттерну картинок слов.

## Область изменений

Только `courses`-функциональность: одна миграция, `courses.js` (2 новых
роута + правка `SELECT`), `imageOptimize.js` (новая функция), `CourseView.jsx`,
`CourseList.jsx`. Ничего в уроках/AI-тренере/Читалке не трогаем.
