## Task 1: SpeechExercise — авто-скролл к результату + заметнее перевод

Файл: `frontend/src/components/SpeechExercise.jsx`.

Контекст: результат ответа (лейбл 🎉/✓/≈/✗, пофонемная подсветка, кнопка
«Далее» вызывающая `onAnswer`) уже реализован (строки ~452-488). Перевод слова
(`displayTranslation`, строки 289-291, 408-410) тоже уже выводится — но мелким
приглушённым текстом внутри блока со словом, что Павел не заметил на телефоне.

Два узких фикса (проверено и подтверждено с владельцем):

1. **Авто-скролл к результату.** Сейчас после ответа блок результата и кнопка
   «Далее» рендерятся, но экран не скроллит к ним — на маленьком экране их не
   видно без ручной прокрутки. В `frontend/src/components/MultipleChoice.jsx`
   есть готовый паттерн: `resultRef` (useRef) на контейнер результата +
   `onReactionEnd={() => setTimeout(() => resultRef.current?.scrollIntoView({ behavior: 'smooth', block: 'end' }), 60)}`
   передаётся в `AvatarReaction`. В SpeechExercise результат/реакция устроены
   иначе — используется `PabloCircle` с `<video onEnded={...}>` (строки 353-358),
   без callback `onReactionEnd`. Добавь аналогичный скролл: заведи `resultRef`
   на блок результата (строки ~453-488) и вызови `scrollIntoView` в подходящий
   момент — когда видео-реакция заканчивается (`onEnded`/`onError` на video,
   строка 356) или, если видео не показывается (`!reactionClip` → `WordImage`),
   сразу при появлении `result` (в `handleResult`/через `useEffect` по `phase === 'result'`).
   Не трогай логику распознавания речи (`useSpeechRecognition`, `handleResult`,
   `phonemeMatch`, TTS/`speak`) — только добавляй скролл.

2. **Заметнее перевод.** Блок «Перевод» (строки 407-410) сейчас
   `fontSize: 15, color: 'var(--ink-soft)'` — сливается с фоном. Сделай его
   визуально заметнее (крупнее и/или контрастнее — по образцу того, как акцент
   расставлен в остальной карточке, например `color: 'var(--ink)'` и/или чуть
   больше `fontSize`, можно с иконкой/подписью «перевод:»). Это просто
   статичная строка текста — НЕ tap-to-translate по словам, ничего кликабельного
   добавлять не нужно. Не меняй сам расчёт `displayTranslation`.

Запрет: не трогай `useSpeechRecognition`, `speak`, `phonemeMatch`,
`germanPhonetic`, `PHONETIC_RULES` — вообще всю логику распознавания/фонетики/TTS.

## Task 2: LangSwitcher — дропдаун с флагами не помещается на мобиле

Файл: `frontend/src/components/LangSwitcher.jsx`.

Используется в двух местах:
- `frontend/src/components/Layout.jsx:449` — `<LangSwitcher pill dropUp />` (сайдбар).
- `frontend/src/components/PublicHeader.jsx:29` — `<LangSwitcher />` (публичные
  страницы, до авторизации).

Проблема: дропдаун (строки 69-100) — `position: absolute`, `right: 0`,
`minWidth: 220`, сетка `repeat(5, 1fr)` на 10 языков. При `right: 0` дропдаун
разворачивается ВЛЕВО от правого края кнопки-триггера. Если кнопка находится
близко к левому краю экрана (сайдбар на мобиле) или окно узкое (~320-360px),
`minWidth: 220` может вылезать за левый/правый край viewport и обрезаться
`overflow: hidden` родителя или просто вылезать за экран.

Сделай так, чтобы дропдаун гарантированно помещался по ширине на мобильном
экране (считай минимальную поддерживаемую ширину ~360px) в ОБОИХ местах
использования (сайдбар с `dropUp`, и публичный хедер без `dropUp`), не ломая
десктопную раскладку. Подход на твоё усмотрение — например: ограничить
`maxWidth` через `calc(100vw - Npx)`, пересчитать `left`/`right` в зависимости
от доступного места, или использовать `position: fixed` с центрированием
относительно кнопки/экрана на узких экранах. Число колонок в сетке при этом
можно уменьшить на мобиле, если понадобится.

Проверь визуально в `npm run dev` на узком вьюпорте (DevTools → мобильная
эмуляция, 360×640 и 320×568) в обоих местах — сайдбар и публичный хедер.

## Task 4: Кнопка «Копировать» в Читалке — только текст на изучаемом языке

Файл: `frontend/src/pages/TextReader.jsx`.

Замечание: `frontend/src/pages/VocabReader.jsx` — это просто обёртка
(`<Vocabulary /> + <TextReader />` рядом), отдельной логики текста там нет —
фикс в `TextReader.jsx` покрывает и VocabReader автоматически. Отдельно менять
`VocabReader.jsx` не нужно.

Чат (`frontend/src/pages/Chat.jsx`, техподдержка/учителю) и AI-тренер — ВНЕ
scope этой задачи (см. Global Constraints — AiTrainer запрещён; Chat.jsx явно
не упоминался владельцем при уточнении и не тянет этот функционал).

Два места рендера текста на изучаемом языке в TextReader.jsx:
- Режим «Читать» (`mode === 'read'`, строки 696-751): `sentences.map(...)`
  (строка 708) — предложение целиком по клику сейчас запускает TTS (`start(i)`,
  строка 710).
- Режим «Двуязычный» (`mode === 'bilingual'`, строки 754+): `bilingual.map(...)`
  (строка 767) — `pair.original` (строка 771, верхний блок) — текст на
  изучаемом языке; `pair.translation` (строка 776, нижний блок) — перевод на
  родной язык.

Сделай кнопку/элемент «Скопировать»: по тапу на одно предложение — копирует
его; должна быть возможность выбрать несколько предложений подряд (по образцу
Telegram — тап включает режим выбора, повторные тапы добавляют/убирают
соседние) и скопировать разом одной кнопкой.

**ВАЖНО (согласовано с владельцем, зафиксировано в IDEAS.md):** копировать
через `navigator.clipboard.writeText` — ТОЛЬКО текст на изучаемом языке
(`sentences[i]` в режиме «Читать»; `pair.original` в режиме «Двуязычный»).
НИКОГДА не копировать `pair.translation` (перевод на родной язык) и не
копировать транскрипцию/произношение.

В режиме «Читать» не ломай существующий клик-по-предложению → TTS (`start(i)`)
— выбор для копирования должен быть отдельным действием (например, отдельная
иконка/чекбокс рядом с предложением, или долгий тап/отдельный "режим выбора"),
а не подменять текущий клик.

## Task 5: Инвентаризация кода → docs/teacher-facts.md

Это НЕ задача на изменение кода — это задача на чтение кода и запись фактов в

домыслов и без "причёсывания".

Пройди по коду и структурируй ответ по разделам:

что конкретно делает ученик (по компоненту в `frontend/src/components/`).

**2. Цикл урока.** Владелец описывает шаги так: «сканируешь → слушаешь слова с
переводом → карточки с ответом → просто карточки → добавь букву → добавь слово
