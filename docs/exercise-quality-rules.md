# Правила корректности упражнений (немецкий) + чек-лист тестов

Свод правил, по которым упражнения должны работать «по правилам немецкого языка», и как
ловить похожие баги. Правила зашиты в промпт генерации (`backend/src/services/claude.js`,
`EXERCISES_PROMPT`) и в проверку ответов на фронте. Этот файл — источник истины и чек-лист.

## 🇩🇪 Правила языка (что генератор и проверка ОБЯЗАНЫ соблюдать)

1. **Заглавная буква.** Все немецкие существительные — с большой буквы (der **H**und, das **A**bendessen).
   Проверка ответа РЕГИСТРОЗАВИСИМА: «hund» ≠ «Hund». Диктант/FillBlank/LetterFill засчитывают
   только верный регистр; при ошибке регистра — подсказка «✍️ Проверь заглавную».
2. **Форма для предложения, не словарная.**
   - Существительное в пропуске — БЕЗ артикля-словарной-формы («Abendessen», не «das Abendessen»).
     Артикль, если нужен по грамматике, остаётся В ПРЕДЛОЖЕНИИ перед пропуском, НЕ дублируется.
   - Глагол — СПРЯЖЁН под подлежащее: «Ich **frage**», «du **fragst**», «wir **fragen**». Инфинитив
     в пропуске после «Ich/Du/Er/Sie» — БАГ.
   - Прилагательное — с правильным окончанием для рода/падежа/числа.
3. **Один определитель.** Нельзя два артикля/детерминатива подряд («der der Papa», «Die die
   Arztvisite»). Фронт-нормализатор (`utils/fillblank.js`) страхует, но генерация не должна их плодить.
4. **Артикль по роду и падежу.** der/die/das — верный род; склонение по падежу (den/dem/des,
   ein/eine/einen…). «mit **dem** Auto», «für **den** Freund».
5. **Верный перевод слова.** translation_ru должен быть точным (grob = грубый, НЕ громкий; laut =
   громкий). Multiple_choice строит варианты из перевода слова — ошибка в переводе = ошибка в упражнении.
6. **Чистый язык.** Предложение — только на изучаемом языке, естественное (A1). Нельзя мешать
   ру/нем в одном предложении и писать мета-пояснения «X значит Y».
7. **Варианты fill_blank — в той же форме, что и ответ.** Все 3 options — либо все голые
   существительные с заглавной, либо все спряжённые глаголы. Правильный вариант в точности = blank.
8. **Предложение полное и осмысленное.** С подставленным blank — грамматически верное и понятное.

## 🔎 Чек-лист «на что ещё проверить тесты» (ловим похожие баги)

Прогонять после генерации/перегенерации или при подозрении. Примеры SQL — на проде
(`docker compose exec db psql -U german_app -d german_learning`).

- **Глаголы-инфинитивы в пропуске** (должны быть спряжены):
  ```sql
  SELECT id, payload->>'sentence', payload->>'blank' FROM exercises
  WHERE type='fill_blank' AND payload->>'sentence' ~ '^(Ich|Du|Er|Sie|Es) ___'
    AND payload->>'blank' ~ 'en$';  -- слово на -en после Ich/Du = вероятный инфинитив
  ```
- **Словарный артикль в пропуске, когда перед ним уже есть определитель** (двойной артикль):
  ```sql
  SELECT id, payload->>'sentence', payload->>'blank' FROM exercises
  WHERE type='fill_blank'
    AND payload->>'blank' ~* '^(der|die|das|den|dem|des|ein|eine|einen|einem|einer) '
    AND payload->>'sentence' ~* '(der|die|das|ein|eine|meine|deine|seine|ihre|zwei|drei) +___';
  ```
- **Существительные со строчной буквы** (нарушение капитализации в данных):
  ```sql
  SELECT id, word_de FROM words WHERE word_de ~ '^[a-zäöü]' AND word_de !~ '^(der|die|das) '
    -- глаголы/прилагательные строчные — норм; сверять существительные вручную
  ```
- **Подозрительные переводы** (ручная сверка спорных слов): смотреть multiple_choice, где
  «правильный» вариант вызывает сомнение (как grob→громкий). Быстрый способ — пройти урок
  и отмечать. Автоматически не ловится — нужен человек/словарь-эталон.
- **Пустые/битые payload** (нет sentence или blank у fill_blank):
  ```sql
  SELECT id, lesson_id FROM exercises WHERE type='fill_blank'
    AND (payload->>'sentence' IS NULL OR payload->>'blank' IS NULL OR payload->>'sentence' NOT LIKE '%___%');
  ```
- **Перевод предложения на локали** (fill_blank без payload_translations → не-русские видят
  предложение без перевода): после генерации всегда прогонять `enrichLesson` (или админ-операцию
  «Упражнения → языки»).

## 🛠️ Как чинить

- **Форма (глагол/артикль)** — только ПЕРЕГЕНЕРАЦИЯ упражнений урока новым промптом
  (`POST /api/lessons/:id/regenerate` = кнопка ⚙️; или скрипт по языку). Сбрасывает SM-2 прогресс урока.
- **Неверный перевод слова** — правка `words.translation_ru` (+`translations`) точечно; затем
  перегенерация урока, чтобы multiple_choice пересобрался из верного перевода.
- **Двойной артикль/регистр в отображении** — уже страхует фронт (`utils/fillblank.js`,
  регистрозависимая проверка), данные править не обязательно.
