# Мини-ТЗ: адаптивное поведение навигации (сайдбар)

## Группировка пунктов меню

Список пунктов не идёт плоским перечнем — разбит на смысловые группы с подзаголовками:

**Сегодня** — отдельно, вне групп, самый верх (домашний экран, самый частый переход).

**Группа «Обучение»** (доступна и ученику, и учителю):

- Уроки  
- Словарь  
- Изучаю  
- Читалка  
- Разговорник

**Группа «Класс»** (только роль `teacher`/`owner`):

- Курсы  
- Ученики  
- Отчёт  
  + Новый урок (действие, визуально выделено акцентным цветом)  
  + Ученик (действие, визуально выделено акцентным цветом)

**Отдельно, перед ADMIN:**

- Справка

**Блок ADMIN** (только `teacher`/`owner`, не рендерится для `student`):

- Загрузка картинок  
- Дополнение словаря  
- Перевод предложений  
- Перевод на все языки  
- Перевод упражнений (с индикатором прогресса)  
- Пересоздать упражнения

Подзаголовки групп — мелкий текст (11px), приглушённый цвет, uppercase, без интерактивности, просто визуальный разделитель. Порядок и состав групп одинаковый на десктопе (в развёрнутом сайдбаре) и в мобильном bottom sheet "Ещё".

## Логика по breakpoint'ам

| Ширина экрана | Поведение сайдбара |
| :---- | :---- |
| ≥ 1024px (десктоп, ноутбук в полный экран) | Сайдбар **всегда развёрнут** — иконка \+ подпись видны постоянно. Пользователь может вручную свернуть его в колонку иконок кнопкой в шапке (состояние сохраняется в localStorage/user preference), но по умолчанию — развёрнут. |
| 640–1023px (ноутбук в половину экрана, планшет альбомная) | Сайдбар **по умолчанию свёрнут** в колонку иконок 60px без подписей. Разворачивается во временную плашку (overlay, не сдвигает контент) по клику на иконку меню или по наведению курсора, закрывается по клику вне области. |
| \< 640px (телефон, планшет портретная) | Сайдбар **полностью скрыт**. Вместо него — нижняя таб-бар на 4–5 пунктов (Сегодня, Уроки, Словарь, Ещё), см. предыдущий макет мобильной навигации. Все остальные пункты меню (Курсы, Изучаю, Ученики, Отчёт, Читалка, Разговорник, Справка, весь блок ADMIN) уходят под пункт "Ещё" — открывается как полноэкранная страница/лист снизу (bottom sheet), не занимает место в таб-баре. |

## Структура пунктов меню (актуальная, из реализованной версии)

Основные (видны в развёрнутом сайдбаре / bottom sheet "Ещё" на мобиле):

1. Сегодня  
2. Курсы  
3. Уроки  
4. Словарь  
5. Изучаю  
6. Ученики  
7. Отчёт  
8. Читалка  
9. Разговорник  
10. Справка

Разделитель, затем действия: 11\. \+ Новый урок 12\. \+ Ученик

Разделитель, затем блок **ADMIN** (виден только роли `owner`/`teacher`, не показывается роли `student`):

- Загрузка картинок  
- Дополнение словаря  
- Перевод предложений  
- Перевод на все языки  
- Перевод упражнений (с индикатором прогресса, напр. "465/2549")  
- Пересоздать упражнения

Внизу — блок настроек (тема светлая/тёмная, голос озвучки, вкл/выкл перевода, язык интерфейса) и кнопка "Выйти" — сворачиваются под одну иконку "Настройки" на мобиле и в свёрнутом десктоп-сайдбаре, чтобы не занимать много места.

## Важные детали реализации

1. **Роль `student` не должна видеть блок ADMIN вообще** (не просто скрыт стилями — не должен рендериться/приходить с бэкенда для этой роли).  
2. **Активный пункт меню** подсвечивается акцентным цветом (золото `#d4a017`) с более тёмной подложкой (`#3a2a00`), как показано в макете.  
3. **Переход между состояниями (развёрнут/свёрнут) должен быть плавным**, но не блокировать контент — на среднем breakpoint (640–1023px) сайдбар всплывает поверх контента (overlay), не толкает его вбок, чтобы не было "прыжка" рабочей области.  
4. **Состояние сайдбара (развёрнут/свёрнут) на десктопе можно запомнить** в localStorage, чтобы при следующем визите пользователь не переключал его заново вручную.  
5. Рабочая область (правая часть) на десктопе не меняет свою внутреннюю вёрстку в зависимости от состояния сайдбара — только доступная ширина.
